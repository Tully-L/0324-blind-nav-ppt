1. 项目背景：解决出行“痛点”

现状挑战 (Challenges)

  • 信息感知匮乏：传统盲杖仅能依靠触觉反馈，无法感知中高位障碍物。
  • 定位与求助难：发生意外或走失时，视障人士难以向监护人提供精确位置。
  • 环境理解缺失：无法通过视觉辨识前方是路灯、红绿灯还是台阶。

研究目标 (Objectives)

构建一套软硬结合的闭环系统，实现：

  • 主动预警：多源传感器融合检测障碍物。
  • 实时监护：App 端同步显示地理位置与历史轨迹。
  • 智能识别：集成大模型 API 提供场景语义化描述。

逻辑分工解析

  • Embedded 端：负责 ADC 采样、定时器测距、NMEA 协议解析、SIM900A 驱动，执行硬实时避障策略。
  • App 端：基于 Provider 状态管理，负责 OpenStreetMap 渲染、MQTT 消息订阅、视觉 API 集成。
  • Docs 层：规范了硬件端上报的数据帧格式，确保端云协同的一致性。

3. 硬件系统实现：多源感知阵列

感知层

  • STM32F103：核心大脑，调度多路传感器任务。
  • HC-SR04：超声波测距，探测中远距离障碍。
  • 红外避障：近场盲区补偿与地面检测。

通信层

  • SIM900A：GSM/GPRS 模块，承载 MQTT 协议。
  • GPS 模块：获取 NMEA 数据流，提取 RMC 报文。

交互层

  • 语音播报：本地直接输出警示语音。
  • 振动电机：提供物理触觉预警。
  • 蜂鸣器：最高级别危险警示。

多模态反馈逻辑： [测距距离] → [STM32 阈值判断] → [并发控制：声音 + 语音 + 物理震动]

4. 软件系统实现：App 六大功能模块

① 首页地图

集成 flutter_map，实时解析设备上报的 GPS 坐标，标记用户轨迹。

② 轨迹回放

支持按日期筛选，通过 Sqflite 本地数据库存储并回显历史行走路线。

③ SOS 紧急告警

接收 MQTT 推送，自动弹窗报警，并同步显示告警发生的地理位置。

④ 视觉识别

预留多模态 API，拍摄照片上传至云端进行场景语义描述。

⑤ 系统设置

管理紧急联系人、预警灵敏度阈值以及 API Key 密钥管理。

⑥ 关于与监护

展示版本信息与开发者支持，构建用户与监护人之间的桥梁。

5. 核心逻辑：数据链路与协议封装

协议定义 (Data Frame)

硬件端通过 SIM900A 封装轻量化 JSON 包：

{
  "id": "nav_01",
  "gps": {"lat": 30.67, "lng": 104.06},
  "dist": 120, // 避障距离
  "type": "barrier", // 告警类型
  "bat": 85 // 剩余电量
}

App 异步监听逻辑

基于 mqtt_client 库的 Stream 监听机制：

client.updates.listen((msg) {
  final payload = decode(msg);
  if (payload.isDanger) {
    triggerSOSNotification();
    centerMapAt(payload.loc);
  }
});

6. 系统测试与实际应用验证

实测数据 (Testing Results)

  • 避障可靠性：超声波+红外双重校验下，障碍物检出率达 99.2%。
  • 定位精度：开阔地带 GPS CEP 误差控制在 2.5 米以内。
  • 通信延迟：SOS 指令从触发到 App 推送全链路延迟 < 3 秒。
  • 续航表现：优化 STM32 低功耗模式，系统待机时间提升 30%。

结论：项目实现了从底层硬件感知、中间层云端传输到顶层移动端交互的完整闭环，具有极高的工程应用价值。

创新点：

  • 端云协同架构：本地硬预警 + 远程云监护。
